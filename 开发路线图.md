# 开发路线图(项目流程)
## 个人任务规划画布

> 配套文档:`PRD.md`
> 面向对象:编程小白
> 原则:每个阶段都有**明确的"完成标准"**,做到了才进下一步;每一步都先理解再写。

---

## 总览:为什么这样排顺序?

第一性原理:**先让画面"看得见",再让它"动起来",最后让它"记得住"。**

```
阶段0 搭好环境  →  阶段1 画出时间轴  →  阶段2 画出卡片
   →  阶段3 拖拽+吸附  →  阶段4 主线串联曲线  →  阶段5 增删改
   →  阶段6 本地保存  →  阶段7 主题打磨  →  阶段8 导出导入
```

每个阶段产出一个"能看到的进步",避免一次写一大坨却跑不起来。

---

## 阶段 0:搭好开发环境(地基)
**目标**:本地能跑起一个空白的 React 项目。

- 安装 Node.js(LTS 版)
- 用 Vite 创建 React + TypeScript 项目
- 安装依赖:`@xyflow/react`、`zustand`
- 跑起开发服务器,浏览器看到默认页面

**完成标准**:浏览器打开本地地址,能看到页面且改动代码会自动刷新。
**你会学到**:前端项目长什么样、怎么启动、依赖是什么。

---

## 阶段 1:画出时间轴(让画面看得见)
**目标**:一张可平移缩放的画布 + 横向日期竖线 + 顶部日期。

- 接入 React Flow,得到一块无边画布(自带平移缩放)
- 渲染一组等距的竖直日期线,顶部标日期
- 深色背景

**完成标准**:能看到 6~7 条日期竖线,可用鼠标拖动平移、滚轮缩放。
**你会学到**:React Flow 的画布、坐标系概念。

---

## 阶段 2:画出小任务卡片(静态)
**目标**:在画布上显示几张写死的圆角矩形卡片,位置对应日期。

- 定义"自定义节点"= 圆角矩形卡片,显示标题
- 用几条假数据,把卡片摆到对应日期的横向位置、对应主线的行
- 暂时不能拖

**完成标准**:画面接近 PRD 里的示意图(两行、多个卡片落在对应日期下)。
**你会学到**:React 组件、自定义节点、数据→画面的映射。

---

## 阶段 3:拖拽 + 吸附到某一天(让它动起来)
**目标**:卡片能横向拖动,松手吸附到最近的一天,日期随之更新。

- 开启卡片拖拽
- 拖动时只允许横向(纵向锁回本行)
- 松手时把 x 坐标换算成最近的日期,更新该卡片的 `date`

**完成标准**:把卡片从 6/30 拖到 7/2,松手后它停在 7/2 正下方,数据里的日期变成 7/2。
**你会学到**:事件处理、坐标与数据的换算、吸附逻辑。

---

## 阶段 4:主线串联曲线
**目标**:同一主线的卡片按时间自动用曲线连起来。

- 同一 `mainTaskId` 的卡片按 `date` 排序
- 相邻两张之间生成一条平滑连线(Edge)
- 拖动卡片后曲线实时重连

**完成标准**:每条主线的卡片被一条曲线依次串起,拖动后曲线跟着变。
**你会学到**:React Flow 连线、由数据自动推导画面。

---

## 阶段 5:增删改(变成真正能用的工具)
**目标**:能自己增删主线、增删小任务、标记完成。

- 新增/删除主线(行)
- 在主线上新增/删除小任务卡片
- 双击卡片编辑标题
- 点击标记完成/未完成(完成有视觉区别)

**完成标准**:不改代码,纯靠界面就能搭出自己的真实任务规划。
**你会学到**:全局状态管理(Zustand)、增删改数据流。

---

## 阶段 6:本地持久化(让它记得住)

**目标**:所有改动自动存进浏览器,刷新不丢。

- 用 Zustand 的 persist 把数据写入 localStorage

**完成标准**:增删改后刷新页面,内容原样还在。
**你会学到**:数据持久化、浏览器存储。

---

## 阶段 7:主题与打磨
**目标**:深色 + 亮绿主题落地,细节顺手。

- 落地 PRD 第 5 节的 CSS 变量主题
- "今天"竖线高亮 + 一键回到今天
- 卡片完成态、悬停态等细节

**完成标准**:整体观感接近预期,深色亮绿、舒服好用。
**你会学到**:CSS 变量主题化、交互细节打磨。

---

## 阶段 8:导出 / 导入(备份与迁移)
**目标**:能把数据导出成 JSON 文件,也能导入回来。

- 导出:把当前数据下载为 `.json`
- 导入:选择 `.json` 文件覆盖当前数据

**完成标准**:导出文件 → 清空 → 导入,数据完整恢复。
**你会学到**:文件下载/读取、数据序列化。

---

## 里程碑速查表

| 里程碑 | 包含阶段 | 达成后你拥有 |
|---|---|---|
| **M1 看得见** | 0~2 | 一张带时间轴和卡片的画布(静态) |
| **M2 能用(MVP)** | 3~6 | 可拖拽、可增删改、刷新不丢的完整工具 |
| **M3 好用** | 7~8 | 主题精致、可备份迁移的成品 |

---

## 给小白的几条原则

1. **一次只推进一个阶段**,跑通了再往下。
2. **每个阶段都先问"为什么这么做"**,理解了再写(第一性原理)。
3. **能看到的进步 > 完美的代码**,先让它跑起来,再慢慢优化。
4. 遇到不懂的概念,随时让我**先解释清楚再继续**。
