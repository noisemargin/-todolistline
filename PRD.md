## 1. 项目背景与目标(Why)

### 1.1 一句话定义
> 一个把**任务分解**(TodoList)和**时间安排**(甘特图)结合在**一张无边画布**上的个人任务规划工具。

### 1.2 解决的问题
传统 TodoList 只能回答"我要做什么",回答不了"什么时候做、做到哪了";
传统甘特图擅长排期,但太重、不适合个人随手规划。
本工具把两者结合:**用拖拽卡片的方式,直观地把"要做的事"摆到"时间线"上**,随时调整。

### 1.3 目标用户
- 主要用户:**你自己**(个人任务规划)。
- 不是团队协作工具,不需要多人、权限、评论等。

### 1.4 设计哲学(第一性原理)
整个产品从**人**出发,只有两个维度:

- **维度一:事情(纵向的分解)** —— 一个人有多条**主线任务**,每条主线拆成多个**小任务**。
- **维度二:时间(横向的流动)** —— 每个小任务对应一个**完成日期**,以"天"为最小单位。

> 把这两个维度叠在一张画布上,就是本产品的全部本质。后面所有功能都必须能追溯回这两个维度,否则就是多余的。

## 2. 核心概念与数据模型(What —— 全文最重要的一节)

### 2.1 三个核心概念

| 概念 | 英文/代号 | 通俗解释 | 视觉表现 |
|---|---|---|---|
| **主线任务** | MainTask | 一件大事,比如"准备考研""做毕业设计" | 画布上**一整行**;它的小任务被一条**曲线串起来** |
| **小任务** | SubTask | 大事拆出来的可执行小步骤 | 一张**圆角矩形卡片**,可横向拖动 |
| **时间轴** | Timeline | 横向流动的日期刻度 | 一组**竖直参考线**,顶部标日期 |

### 2.2 它们的关系(一图说清)

```
时间(左→早,右→晚)
  6/30        7/1        7/2        7/3
   │           │          │          │
   │           │          │          │   ← 每个日期 = 一条竖直参考线
   │           │          │          │
 ┌────┐   ┌────┐                         主线A(第一行)
 │任务1│~~~│任务2│~~~┐
 └────┘   └────┘   │ ┌────┐
                    └─│任务3│
                      └────┘
 ┌────┐   ┌────┐                         主线B(第二行)
 │任务1│~~~│任务2│~~~┐
 └────┘   └────┘   │      ┌────┐
                    └──────│任务3│
                           └────┘
   │           │          │          │
```

要点:
- **横向位置 = 时间**。卡片落在哪个日期区间,就代表它排在那一天。
- **某条日期竖线左边 = 要在那天之前完成**(deadline 语义)。
- **同一主线的卡片用曲线依次串起来**,按时间从左到右(就像穿珠子)。
- **多条主线 = 上下堆叠的多行**。

### 2.3 数据结构(用 JSON 描述,这是程序真正存的东西)

```jsonc
{
  "version": 1,
  "mainTasks": [
    {
      "id": "m1",            // 唯一标识
      "title": "准备考研",    // 主线名字
      "color": "#39FF6A",    // 这条线的颜色(用于区分多条主线)
      "order": 0,            // 第几行(0 在最上面)
      "createdAt": "2026-06-30"
    }
  ],
  "subTasks": [
    {
      "id": "s1",
      "mainTaskId": "m1",    // 属于哪条主线
      "title": "看完第一章",
      "date": "2026-06-30",  // 排在哪一天(最小单位:天)
      "done": false,         // 是否完成
      "createdAt": "2026-06-30"
    }
  ]
}
```

> 设计取舍:小任务的**横向位置由 `date` 决定**,不单独存 x 坐标 —— 这样"拖到 7/1 下面"就等于"把 date 改成 7/1",数据和画面永远一致,不会错位。
> 串联曲线**不存数据**,而是由程序"同一主线的卡片按 date 排序后自动连线"算出来 —— 少存一份数据就少一处出错。

## 3. 核心交互与视图(How it looks)

### 3.1 画布(Canvas)
- **无边画布**:可以无限平移(拖空白处)、缩放(滚轮)。
- 内容默认居中显示"今天"附近的日期。

### 3.2 时间轴(Timeline)
- 横向排列的**日期竖线**,顶部标注日期(如 6/30、7/1)。
- 间距可以**不均分**(MVP 先做等距,后期再做自由间距)。
- "今天"这条线高亮(用主题亮绿色)。

### 3.3 主线任务(行)
- 每条主线占一整行,行首显示主线名字和颜色。
- 可新增/重命名/删除主线;可调整行的上下顺序。

### 3.4 小任务卡片(Card)
- **圆角矩形**,显示任务标题。
- **可横向拖动**在天与天之间;松手后**吸附**到最近的某一天,`date` 随之更新。
- 完成的卡片有"已完成"样式(如打勾 / 变暗 / 划线)。
- 双击或点击可编辑标题、删除。

### 3.5 串联曲线(主线的"线")
- 自动把**同一主线**的卡片按时间顺序用平滑曲线连起来。
- 拖动卡片改变顺序时,曲线实时重连。

## 4. 功能清单(按优先级分期,防止一口吃成胖子)

> 用 MoSCoW 思路:P0 = 必须有(MVP),P1 = 应该有,P2 = 以后再说。

### P0 —— MVP(先把核心闭环跑通)
- [ ] 无边画布:平移、缩放
- [ ] 横向时间轴:渲染日期竖线 + 顶部日期标签(等距即可)
- [ ] 新增/删除**主线任务**(行)
- [ ] 在某条主线上新增/删除**小任务卡片**
- [ ] 卡片**横向拖拽**,松手吸附到某一天并更新日期
- [ ] 同一主线卡片**自动串联曲线**
- [ ] 标记小任务**完成/未完成**
- [ ] 数据**自动保存到浏览器**(刷新不丢)
- [ ] 深色模式 + 亮绿主题色

### P1 —— 第二期(更好用)
- [ ] 编辑卡片标题、主线名字、颜色
- [ ] 调整主线行的上下顺序(拖拽排序)
- [ ] "今天"竖线高亮 + 一键回到今天
- [ ] 导出 / 导入 JSON(备份、换设备)
- [ ] 卡片小细节:备注、所属时间提示

### P2 —— 以后(锦上添花)
- [ ] 日期竖线**自由间距**(非均分)
- [ ] "deadline 卡片"(明确放在某日期线左侧表示截止)
- [ ] 进度统计(每条主线完成百分比)
- [ ] 浅色模式 / 主题色可切换
- [ ] 撤销 / 重做

## 5. 视觉与主题

### 5.1 基调
- **深色模式优先**。
- **主题色:亮绿色**(示意 `#39FF6A`,开发时再微调)。

### 5.2 设计 Token(用 CSS 变量统一管理,改一处全局生效)

```css
:root {
  --bg-canvas:    #0E0F12;   /* 画布背景(深) */
  --bg-card:      #1A1C20;   /* 卡片背景 */
  --line-date:    #2A2D33;   /* 日期竖线 */
  --line-today:   #39FF6A;   /* 今天高亮 */
  --accent:       #39FF6A;   /* 主题亮绿 */
  --text-primary: #E8EAED;   /* 主文字 */
  --text-muted:   #8A8F98;   /* 次要文字 */
  --task-line:    #39FF6A;   /* 主线串联曲线 */
}
```

## 6. 技术选型与理由

| 部分 | 选择 | 为什么(给小白) |
|---|---|---|
| 构建工具 | **Vite** | 启动快、配置少,现代前端入门首选 |
| 框架 | **React + TypeScript** | 资料最多、生态最大;TS 能在写错时提前报错,配 AI 更顺 |
| 画布/拖拽 | **React Flow (@xyflow/react)** | 无边画布、平移缩放、拖拽节点、节点连线**全都自带**,正中需求,省掉最难的部分 |
| 状态管理 | **Zustand** | 极简,几行代码就能管全局数据,小白友好 |
| 本地存储 | **Zustand persist → localStorage** | 自动把数据存进浏览器,刷新不丢,零后端 |
| 样式 | **CSS 变量 + CSS Module** | 主题化清晰,顺便学样式底层 |

> 概念映射(帮助理解为什么 React Flow 这么合适):
> - 小任务卡片 = React Flow 的**自定义节点 (Node)**
> - 主线串联曲线 = React Flow 的**连线 (Edge)**
> - 日期竖线 = 画布的**背景层 / 自定义层**
> - 平移缩放 = React Flow **自带**

## 7. 非目标(明确不做,防止范围膨胀)

- ❌ 不做多人协作、登录、账号、云同步(单机单人)。
- ❌ 不做后端服务器、数据库(数据只存浏览器)。
- ❌ 不做手机 App / 小程序(先做网页版)。
- ❌ 不做精确到小时/分钟的排期(最小单位是"天")。
- ❌ MVP 不做浅色模式(深色优先)。

## 8. 开放问题 / 待确认

> 这些不影响开始 MVP,后续遇到再定。

1. 一条主线的小任务能否在**同一天**放多个?(倾向:可以,同一天纵向略微错开)
2. 卡片吸附是"吸到日期线"还是"吸到两线之间的区间"?(倾向:吸到区间 = 那一天)
3. 主线行高、卡片宽度等具体尺寸 —— 留到做视觉时定。
