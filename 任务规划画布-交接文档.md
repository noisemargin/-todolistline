## 1. 项目概览

**个人任务规划画布** — 把 TodoList 的任务分解和甘特图的时间轴结合在一张无边画布上。单机单人、纯前端、深色模式。

**技术栈**: Vite 8 + React 19 + TypeScript 6 + React Flow 12(@xyflow/react)+ Zustand 5

**开发服务器**: `npm run dev` → `http://localhost:5173/`

---

## 3. 数据模型(唯一数据源 = Zustand store)

### taskStore.ts(持久化到 localStorage,key=`task-canvas-store`)

```ts
interface MainTask {
  id: string;       // "id-102-1751832000000"
  title: string;    // "考研英语"
  color: string;    // "hsl(145, 55%, 60%)" — HSL 统一明度,只变色相
}

interface SubTask {
  id: string;
  mainTaskId: string;  // 属于哪条主线
  title: string;
  date: number;        // UTC 毫秒时间戳(JSON 友好,序列化不丢)
  done: boolean;
  y?: number;          // 画布上自由纵向位置;不存时自动按主线行计算
}
```

**关键操作**:
- `addMainTask(title)` → 同时创建该主线的第一张卡片(标题=主线名,排在今天)
- `addSubTask(mainTaskId, date?)` → 新建卡片;date 不传默认今天 9am
- `deleteSubTask(id)` → 删卡片;若该主线无剩余卡片,连主线一并删除
- `moveSubTask(id, date, y)` → 更新卡片日期和纵向位置
- `renameMainTask`/`renameSubTask`/`toggleSubTaskDone`/`deleteMainTask`

### uiStore.ts(不持久化,刷新重置)

```ts
editMode: boolean;           // false=展示, true=编辑
editingCardId: string|null;  // 当前正在内联改名的卡片
selectedIds: string[];       // 框选中的卡片(Delete 删除)
```

## 4. 核心交互模型(右下角「✎ 编辑」切换)

### 展示模式(editMode=false) — 日常看板

| 操作 | 行为 |
|------|------|
| 左键拖卡片 | 自由拖拽,松手 x 吸附到最近一天,y 自由 |
| 双击卡片 | 切换完成/未完成(变暗+删除线) |
| 中键拖空白 | 平移画布 |
| 滚轮 | 缩放 |
| 「● 今天」按钮 | 平滑动画回到今天 |

### 编辑模式(editMode=true) — 规划调整

| 操作 | 行为 |
|------|------|
| 左键拖卡片 | 自由拖拽(同展示模式) |
| **单击卡片** | 进入内联改名(220ms 延时,防误触) |
| 悬停卡片**左侧** | 出现"+",点=前一天新建同主线卡片 |
| 悬停卡片**右侧** | 出现"+",点=后一天新建同主线卡片 |
| 底部输入栏 | 打字+Enter/新增 → 新建主线(自动带第一张标题卡) |
| 左键拖拽框选 | 从**左→右**=完全框住才选中;**右→左**=碰到即选整条主线 |
| Delete/Backspace | 删除所有选中的卡片 |
| 悬停串联曲线 | 线中点出现"删除主线"按钮 |

### 两种模式都可用

| 操作 | 行为 |
|------|------|
| 「↓ 导出」 | 下载 `task-canvas-日期.json` |
| 「↑ 导入」 | 选择 JSON 文件恢复数据 |

## 5. 关键实现细节(防踩坑)

### Canvas.tsx — 节点/连线派生流程
1. `dateNodes`(useMemo,[])→ 日期竖线,静态,只算一次
2. `cardNodes`(useMemo,[mainTasks,subTasks])→ 从 store 派生,`isFirst` 标记每条主线最早卡
3. `initEdges`(useMemo,[mainTasks,subTasks])→ 同主线卡片按 date 排序相邻连线,`data.mainTaskId` 传给边
4. `useEffect` 把 dateNodes+cardNodes 同步到 `useNodesState`;initEdges 同步到 `useEdgesState`

### CardNode.tsx — 关键细节
- **标题卡(isFirst=true)**:实心纯色底色+深色文字+fontWeight 700,用作主线标题
- **普通卡(isFirst=false)**:半透明底色+彩色边框
- 左右"+"热区:`.card-zone` 宽 44px 在卡片外侧,悬停才显示按钮,不会因鼠标移出卡片而消失
- 单击/双击区分:click 延时 220ms,双击取消计时器
- 选中态:`outline: 2px solid var(--accent)`

### TaskLineEdge.tsx — 自定义边
- 使用 `getBezierPath` 画贝塞尔曲线(curvature=0.18)
- 编辑模式下加 20px 宽透明命中区用于悬停检测
- 悬停时在 `EdgeLabelRenderer` 显示"删除主线"按钮
- `data.mainTaskId` 通过 edge 的 data 字段传入

### SelectionLayer.tsx — 框选
- 监听 `.react-flow__pane` 的 mousedown→ 画选择框
- 方向判断:`dx >= 0` = 左→右(window),`dx < 0` = 右→左(crossing+扩展到整条主线)
- 坐标换算:`screenToFlowPosition` 转屏幕像素到画布坐标
- Delete 键监听:排除 input/textarea 内的按键

### DateLineNode.tsx — 日期线
- 线宽= `(isToday?2:1) / zoom`,缩放时粗细恒定
- 月初有"7月"标记,年初有"2027"标记
- today 线带 `today-glow` class(CSS box-shadow 微光)

### vite.config.ts — 关键配置
- `resolve.dedupe: ['react','react-dom']` — **必须保留**,否则 React Flow 触发 "Invalid hook call"

## 6. 颜色体系

**主题 Token**(theme.css):
- `--accent` = `#39FF6A`(亮绿,用于模式切换按钮、月初标记、选中态描边)
- `--line-today` = `#BCC0C8`(近白灰)
- `--line-date` = `#343840`(弱化灰)

**主线颜色**(taskStore.ts HUES 数组):
- 7 个色相:`[145, 215, 275, 35, 355, 185, 305]`
- 统一格式:`hsl(H, 55%, 60%)` — 固定明度和饱和度,只变色相,视觉重量一致
- 新建主线时按顺序轮换

## 8. 给接手 agent 的实操提示

1. **先跑起来**: `npm run dev`,打开 localhost:5173,硬刷新确认画面正常
2. **理解数据流**: 所有改数据的操作都必须经过 taskStore 的 action,不要直接改 nodes/edges
3. **不要删 `resolve.dedupe`** 在 vite.config.ts — 删了 React Flow 崩溃
4. **框选与拖拽的边界**: 卡片有 `draggable:true`,空白处左键画框,中键平移 — 三者互不冲突
5. **切换到编辑模式看看**: 底部输入栏、左右"+"、框选删除 — 大部分功能只在编辑模式暴露
